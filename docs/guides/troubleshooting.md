# Troubleshooting

This guide covers the most common errors encountered when setting up or running the
pytorch-lab project, with exact fixes.

---

## Quick reference table

| Symptom | Likely cause | Fix |
|---|---|---|
| `NotImplementedError: The operator ... is not implemented for MPS` | MPS fallback not enabled | Set `PYTORCH_ENABLE_MPS_FALLBACK=1` |
| `MPS available: False` | Not running on Apple Silicon | Only MPS and CPU are supported; see note below |
| `ModuleNotFoundError: No module named 'going_modular'` | Project not installed | Run `uv sync` from the repo root |
| `No such file or directory: scratch/datasets/...` | Dataset not fetched | Run the correct fetch target (see table below) |
| `FileNotFoundError: models/ScreenNet*.pth` | Weights not present | Train first or download pre-trained weights |
| `uv: command not found` | uv not installed | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| `ruff check` reports errors | Lint violations | `uv run ruff check . --fix` or fix manually |
| `pyright` reports type errors | Type violations | Read the error message; missing imports can often be suppressed with `# type: ignore` |
| `pytest` can't find tests | Wrong working directory | Run `make test` from the repo root |
| TensorBoard shows no data | Wrong `--logdir` | Make sure you're in the subproject directory when running `make tensorboard` |
| `pip install` or `conda install` used | Wrong package manager | Undo and use `uv add` instead; then `uv sync` |

## Detailed fixes

### MPS operator not implemented

**Error:**

```
NotImplementedError: The operator 'aten::...' is not implemented for MPS.
Set the environment variable PYTORCH_ENABLE_MPS_FALLBACK=1 to use the CPU
fallback for this op.
```

**Fix:**

```bash
export PYTORCH_ENABLE_MPS_FALLBACK=1
```

To make this permanent:

```bash
echo 'export PYTORCH_ENABLE_MPS_FALLBACK=1' >> ~/.zshrc
source ~/.zshrc
```

---

### MPS not available

**Symptom:** `make env-works` prints `MPS available: False`.

**Explanation:** This project is designed for Apple Silicon Macs (M1/M2/M3/M4). The
MPS backend is not available on Intel Macs or in virtual machines without GPU
passthrough. Training will automatically fall back to CPU, which is significantly
slower but functionally correct.

**Check your chip:**

```bash
sysctl -n machdep.cpu.brand_string
```

If the output does not contain "Apple M", you are on Intel and MPS is not available.
CPU training still works — it just takes longer.

---

### `ModuleNotFoundError: No module named 'going_modular'`

**Cause:** The `going_modular` package has not been installed into the virtualenv, or
you are running Python from outside the `uv`-managed environment.

**Fix:** Always run commands through `uv run` or re-sync the environment:

```bash
uv sync
```

Never activate the virtualenv manually and run `python main.py` directly — use
`uv run python main.py` instead, which ensures the correct environment is used.

---

### Missing classification dataset

**Error:**

```
FileNotFoundError: scratch/datasets/twitter_facebook_tiktok/train/ not found
```

**Fix:** Fetch and unzip the dataset from the repo root:

```bash
make download-dataset
make unzip-dataset
```

---

### Missing localization dataset

**Error:**

```
FileNotFoundError: scratch/datasets/twitter_screenshots_localization_dataset/ not found
```

**Fix:**

```bash
make fetch-assets
# or, for the dataset only:
make download-localization-dataset
```

---

### Missing model weights

**Error:**

```
FileNotFoundError: models/ScreenCropNetV1_378_epochs.pth not found
```

**Fix (screencropnet):** Either train from scratch or download the pre-trained weights:

```bash
cd screencropnet
make get-best-model
```

Or fetch all assets at once from the repo root:

```bash
make fetch-assets
```

**Fix (screennet):** The screennet weights are only generated by training. Run:

```bash
cd screennet
make train
```

---

### `uv` is not found

**Error:**

```
zsh: command not found: uv
```

**Fix:**

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Then reload your shell:

```bash
source ~/.zshrc
```

---

### Ruff lint failure

**Symptom:** `make lint` or `make check` exits non-zero with ruff diagnostics.

**Fix:** Run the auto-fixer for safe fixes:

```bash
uv run ruff check . --fix
```

For errors that cannot be auto-fixed (e.g. unused variables, logic issues), read the
error message and fix manually. The ruff rule code in the output (e.g. `F841`) can be
looked up in the [ruff documentation](https://docs.astral.sh/ruff/rules/).

---

### Pyright type errors

**Symptom:** `make typecheck` exits non-zero.

**Common causes and fixes:**

| Error pattern | Fix |
|---|---|
| `reportMissingModuleSource` on torch internals | Already disabled in `pyproject.toml`; if you see it, run `uv sync` |
| `Cannot access attribute "..."` on a torch tensor | Add `# type: ignore` on that line as a last resort |
| `Module "X" has no exported attribute "Y"` | The import may be wrong; verify against the library's public API |

---

### TensorBoard shows no scalar data

**Symptom:** TensorBoard loads but the Scalars tab is empty.

**Causes and fixes:**

1. You ran `make tensorboard` from the repo root instead of from inside the subproject
   directory. The `runs/` logdir is relative to the subproject. Fix:

   ```bash
   cd screennet   # or screencropnet
   make tensorboard
   ```

2. Training was run without `--to-disk`. The `make train` targets already include
   `--to-disk`, but if you called `main.py` manually and omitted it, TensorBoard events
   may not have been written.

---

### pytest cannot discover tests

**Symptom:** `uv run pytest` reports `no tests ran` or `ERROR collecting`.

**Fix:** Ensure you are in the repo root when running pytest. The `testpaths = ["tests"]`
setting in `pyproject.toml` resolves relative to the root:

```bash
cd /path/to/pytorch-lab
make test
```

---

### `inv` command not found

**Symptom:**

```
zsh: command not found: inv
```

**Fix:** The `invoke` package is a dev dependency. Install it:

```bash
uv sync
```

Then use it through uv if needed:

```bash
uv run inv local.clean
```
